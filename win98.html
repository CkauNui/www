Windows 98

Коллекция оформлений Windows 98 (win98) с русским описанием, обложками, логотипами, видео, скриншотами и фан-артами.

Данная система работает на RetroArch ядре "DOSBox-Pure".
Сюда вошли тайтлы, выпускавшиеся примерно с 1997 по 2001 год.

Рекомендации, проблемы и решения

Scroll Lock - вкл/выкл режим фокусировки (отключает хоткеи и заставляет физическую клавиатуру работать полностью).
Нажатие на левой стик - вызвать экранную клавиатуру.

- Установлен драйвер 3dfx Voodoo, последний совместимый DirectX 8.1, необходимые кодеки и библиотеки.
- Сохранены и настроены все требуемые пути в реестре для запуска тайтлов и обнаружения CD без их установки.
- Для экономии места файлы с CD были обрезаны, оставлены только те, что нужны для запуска и воспроизведения музыки/видео.
- Добавлены .opt файлы с настройками RetroArch.
- Автозапуск тайтлов.
- (Не обязательно) мои настройки большинства тайтлов.
- Работает сетевая игра (через NetPlay). Установлен сетевой адаптер: IP хоста всегда будет 192.168.0.1, а IP других пользователей будут оканчиваться на 0.2, 0.3 и т. д. (удобно для подключения по TCP/IP).
- Файлы, названные кириллицей, переименованы транслитом (некоторые из них имели некорректную кодировку, из-за чего не отображались и не работали — например, нельзя было выбрать миссию).
- Кастомные обложки (600x900), скриншоты (480p), видео (480p), логотипы (600x300) и фан-арты.
- Для мануалов сделал "вырезки" из журналов тех лет (85шт).
- Проставлены все метаданные.

Краткое описание системы

Windows 98 - операционная система, ориентированная на потребителя, разработанная компанией Microsoft в рамках семейства операционных систем Microsoft Windows 9x. Вторая операционная система в линейке 9x, она является преемницей Windows 95 и была выпущена в производство 15 мая 1998 года, а в розничную продажу - 25 июня 1998 года. Как и предшественница, она представляет собой гибридный 16- и 32-разрядный монолитный продукт с этапом загрузки, основанным на MS-DOS.

Windows 98 - это веб-интегрированная операционная система, имеющая много общего со своей предшественницей. Большинство ее усовершенствований носят косметический характер и направлены на улучшение пользовательского интерфейса, но есть и ряд функций, направленных на расширение функциональности и возможностей системы, в том числе улучшенная поддержка и доступность USB, а также поддержка аппаратных новинок, таких как DVD-плееры.

Установка системы (для Batocera)

В раздаче лежит архив с папками и файлами, которые нужно скинуть на вашу флешку\жд по пути "\\batocera\share". Таким образом данная система появится и не затрётся в случае обновления Batocera.

По итогу файл "es_systems_win98.cfg" должен лежать в "/userdata/system/configs/emulationstation/"
(он же "\\batocera\share\system\configs\emulationstation\")
А ".opt" файлы в "/userdata/system/configs/retroarch/config/DOSBox-pure/"
(он же "\\batocera\share\system\configs\retroarch\config\DOSBox-pure\")

Установка системы (для RetroBat)

В раздаче лежит архив с папками и файлами, которые нужно скинуть на ваш жд в папку "RetroBat". Таким образом данная система появится в списке и не затрётся в случае обновления RetroBat.

По итогу файл "es_systems_win98.cfg" должен лежать в "RetroBat\emulationstation\.emulationstation\"
А ".opt" файлы в "RetroBat\emulators\retroarch\config\DOSBox-pure\"

Общая установка и настройка

Остальные папки нужно скинуть на вашу флешку\жд по пути:
[Retrobat] В папку "\Retrobat\"
[Batocera] В папку "\\batocera\share"

Папка "bios" содержит основной дистрибутив Windows 98 SE.img.
Скопируйте этот файл в вашу папку bios.

Папка "roms\win98" содержит медиа и файл gamelist.xml.
Скопируйте папку win98 в вашу папку roms.

Папка "roms - Распаковать RAR архивы" содержит .rar архивы (сжаты из-за большого количества файлов), которые нужно распаковать, а полученные папки перенести в "roms\win98".
Пример: После распаковки Age of Empires Gold Edition (AoE.pc).rar появится папка AoE.pc — её нужно переместить в roms\win98.
п.с. Эмулятор может работать с ZIP архивами, но некоторые тайтлы могут случайно вылетать с ошибкой чтения файлов.

Папка "saves" содержит контейнеры вида Название-CDRIVE.sav.
Эти файлы создаются при включённой опции Save Difference Per Content (включил в .opt файлах), она сохраняет все изменения на диске C (рабочий стол, ярлыки, автозагрузка и т. д.), не затрагивая основной дистрибутив Windows 98 SE.img.
Основная система Windows 98 SE.img остаётся нетронутой, а пользовательские изменения хранятся в Название-CDRIVE.sav.
Важно: не отключайте опцию Save Difference Per Content и не вносите изменения без неё. Иначе данные могут записаться в основной образ Windows 98 SE.img, и все тайтлы, использующие Save Difference Per Content, перестанут работать. Если такое произошло - замените Windows 98 SE.img на оригинальный файл из раздачи.

Папка "saves - Не обязательные настройки и фиксы (нужно переименовать)" содержит дополнительные авторские настройки (о них чуть ниже отдельно).

Папка "saves - Не обязательные настройки и фиксы"

Эти настройки не обязательны, но крайне желательны. Большинство тайтлов уже настроены, но при запуске система создаёт для каждого пользователя свой ID (например, Allods1-BB58E885.sav). От чего он зависит, я так и не разобрался. Даже если перенести файлы на другой диск, ID изменится.

Чтобы применить настройки, нужно переименовать файлы:
1. Откройте папку \saves\win98. В ней вы увидите файлы двух типов:
• "Название-ID.sav" (например, Allods1-BB58E885.sav — это мой файл). Его нужно переименовать в ваш.
Этот файл сохраняет все изменения на диске D (то есть в папке с тайтлом).
• "Название-CDRIVE.sav" (например, Allods1-CDRIVE.sav). Его переименовывать не нужно.
Этот файл создаётся при включённой опции Save Difference Per Content и сохраняет все изменения на диске C (рабочий стол, ярлыки, автозагрузку и т. д.), не затрагивая основной образ Windows 98 SE.img в папке bios.
Важно: Не отключайте Save Difference Per Content, иначе изменения на диске C могут сломать систему, и тайтлы перестанут запускаться.

2. Запустите и закройте тайтл. В папке \saves\win98 появится новый файл, например, Allods1-E75996DB.sav (ваш ID).

3. Переименуйте старый файл в новый:
• Скопируйте название нового файла (Allods1-E75996DB.sav).
• Удалите новый файл.
• Переименуйте старый файл (Allods1-BB58E885.sav) в новое название (Allods1-E75996DB.sav).

После этого настройки должны примениться. Для проверки на диске D должен появиться ярлык рядом с папкой Game.

Рекомендации, проблемы и решения

При запуске может открыться меню DOSBox Pure. В этом случае:
1. Перейдите в пункт [Run Installed Operating System].
2. Выберите Windows 98 SE.
3. Чтобы в следующий раз пропускать это меню, нажмите один раз "стрелку вправо" (на клавиатуре или геймпаде).
Рядом с Windows 98 SE должна появиться пометка [Set Auto Start].
В раздаче это уже настроено, но всякое может быть.

Включение 3dfx Hardware OpenGL, повышение производительности

1. Проверяем включен ли драйвер 3dfx
Hotkey+B > Опции ядра > Video > 3dfx Voodoo Emulation - Enabled - 12MB memory

2. В настройках RetroArch включаем поддержку 3dfx Hardware OpenGL, что значительно повысит производительность в 3D-тайтлах.
Hotkey+B > Опции ядра > Video > 3dfx Voodoo Performance - Hardware OpenGL

Если такой опции нет, то ваше ядро пока не обновилось, ждите обновления Batocera/Retrobat или замените ядро вручную.

Scale и разрешение

По умолчанию в конфигах (.opt файлах) поставил Scale 2x, но вы можете установить его выше или ниже в настройках RetroArch.
Hotkey+B > Опции ядра > Video > 3dfx Voodoo OpenGL Scaling - 2x (я обычно использую 4x)

Лучше не менять разрешение внутри приложения (оставить 640х480), а поменять Scale в настройках RetroArch.

Картинка с 3DFX выглядит слишком тёмной / светлой

Hotkey+B > Опции ядра > Video > 3dfx Voodoo Gamma Correction - подобрать под себя

Примерные настройки в .opt файлах (в настройках RetroArch)

Если не используете подготовленные .opt файлы или просто для ознакомления
Hotkey+B > Опции ядра > Video > 3dfx Voodoo Emulation - Enabled - 12MB memory
Hotkey+B > Опции ядра > Video > 3dfx Voodoo Performance - Hardware OpenGL
Hotkey+B > Опции ядра > Video > 3dfx Voodoo OpenGL Scaling - 4x
Hotkey+B > Опции ядра > Video > SVGA Mode - S3 Trio64
Hotkey+B > Опции ядра > Video > SVGA Memory - 4MB
Hotkey+B > Опции ядра > Video > Aspect Ratio Correction - On

Hotkey+B > Опции ядра > Performance > Emulated Performance - MAX (если игра идёт слишком быстро, то выбрать Pentium III или другое)

Hotkey+B > Опции ядра > System > Memory Size - 128 MB (или по вашему усмотрению). Если всё начинает сильно тормозить, то стоит убавить значение.
Hotkey+B > Опции ядра > System > Advanced - OS Disc Modifications > Save Difference Per Content

Управление геймпадом в Windows

Правый стик → Движение курсора мыши
R (Правый триггер) → Правая кнопка мыши
L (Левый триггер) → Левая кнопка мыши
Крестовина / Левый стик → Клавиши-стрелки (↑ ↓ ← →)
Start → Enter
Select → Esc
Нажатие на левый стик (L3) → Открыть экранную клавиатуру

Поворот камеры на геймпаде

Если играете на геймпаде, для возможности "крутить камеру" на правый стик, нужно поменять "Тип устройства" на "Mouse /w Right Analog"
[RetroArch] Hotkey+B > Управление > Привязки порта 1 > Тип устройства - Mouse /w Right Analog. Ниже можно настроить управление.

[Retrobat] Select > Доп. настройки системы > Emulation > Controlls > Player 1 Conntroller Type - Keyboard + Mouse (Right An.). Если играете на геймпаде, даёт возможность "крутить камеру" на правый стик, остальные настройки управления делаются через меню RetroArch.

Если система сломалась, перестало запускаться

Проверьте, включена ли опция Save Difference Per Content.
Замените Windows 98 SE.img на оригинальный файл из раздачи.

Добавление своих тайтлов

Если хотите добавить свой тайтл, лучше всего:
1. Скопировать конфиг (например, Allods1.opt из папки \retroarch\config\DOSBox-pure).
2. Переименовать копию (например, в ReEvil1.opt).

Важно!
• Опция Save Difference Per Content должна быть включена (как в моих .opt файлах). Не вносите изменения без неё. Иначе данные могут записаться в основной образ Windows 98 SE.img, и все тайтлы, использующие Save Difference Per Content, перестанут работать.
